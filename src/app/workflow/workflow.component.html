Website Components and Costs

To have your website up and running you will need three components:

Domain

Name for your website which is also called an URL. For example www.example.com

Hosting

Hosting provider is a fancy name for an online place where your website is stored and processed.

Code

Website itself which is files with the code.

Domain

Google Domains is one of the services that let you register your website’s name. The prices start from $12 per year.

Choose your domain

Hosting

Hosting pricing is a little bit more complex.

It depends on the website and on how many users visit it. It can go up to several hundred. Do not be scared by this cost. You would have to have an extremely complex website with thousands of visitors, in order for the price to reach that number.

For example, on Heroku (hosting that I prefer and use) simple website can be hosted on a basic plan for just $7 per month. Moreover, since you pay only for what you use, it might be even less than $7.

Development

Price of the development can vary from $300 to $1500 depending on the complexity and functionality of your website. Below you can find an article in which I tried to categorize websites by complexity.

I stop at $1500 because, in my opinion, a website that has such a sophisticated functionality and requires more development time and effort should, most certainly, be maintained constantly. Which means you might consider hiring a permanent web developer for your business.

Websites Categories

The cost and development time depends on the complexity of your website. I described some of the common types of websites. This way you can plan your budget.

Informational website

Purpose: The purpose of an informational website is simply to tell the world about you and your business.

Description: It might be just a one page website that contains description of your services and how to contact you. Information on such websites does not have to be changed very often and usually it is done inside the code.

Complexity: This is the most simple type of websites that does not require a database or constant maintenance.

Time: It can be done in 3-7 days depending on the amount of information and design.

Cost: $75-150

See Example

Dynamic content website

Purpose: Systematically provide users with new information.

Description: Contents of dynamic websites is updated often: writing new articles, posting new recipes, uploading new pictures. Websites like that have a database where all the content is stored and retrieved from. They also have a login system with at least one user (admin) who can login and create/edit/delete the content.

Complexity: Dynamic websites include database and user login system. It takes a little bit more time to set everything up.

Time: You can expect 1-2 weeks of work and 3-7 days of testing before your site can go live.

Cost: $300-600

See Example

Interactive website

Purpose: Process users’ requests.

Description: Interactive website is an advanced dynamic content website. The difference is that interactive websites do not only provide information, but also let your customers perform some tasks such as request and schedule services, post their own articles, leave comments, etc.

Complexity: Interactive websites have more complex login system that gives different privileges to different users. For example, visitors can add posts and leave comments, but only admin can manage the entire content of the website.

Time: The development period depends greatly on the website functionality. Roughly, it will take 2-4 weeks to build an interactive website and one week to test it.

Cost: $600

See Example

E-Commerce

Purpose: Sell you products to customers.

Description: It has login system and privileges as an interactive website plus it should perform financial transactions. Your users should be able to buy your products and pay for it online.

Complexity: As for now, I am not working on e-commerce projects. E-commerce websites require a higher level of security and continuous maintenance. If you are planning an ecommerce business you might consider hiring a permanent developer.

There is an exception: if your stock contains just a few products and you are going to use only PayPal as a payment method. Websites with PayPal integration do not require additional security methods because all transactions are happening on the third party website (PayPal).
